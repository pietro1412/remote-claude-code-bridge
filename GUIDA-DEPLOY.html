1. Installer Automatico

L'installer automatico installa tutti i prerequisiti, clona il progetto, lo builda, avvia il server e genera il token di accesso. Un solo comando e sei operativo.

Linux (Ubuntu/Debian/Fedora/Arch)

Apri un terminale sul PC fisso ed esegui:

curl -fsSL https://raw.githubusercontent.com/pietro1412/remote-claude-code-bridge/master/scripts/install.sh | bash

Oppure, se preferisci leggere lo script prima:

curl -O https://raw.githubusercontent.com/pietro1412/remote-claude-code-bridge/master/scripts/install.sh
cat install.sh   # leggi lo script
bash install.sh  # eseguilo

Windows

Apri PowerShell come Amministratore ed esegui:

irm https://raw.githubusercontent.com/pietro1412/remote-claude-code-bridge/master/scripts/install.ps1 | iex

Oppure scarica ed esegui manualmente:

Invoke-WebRequest -Uri "https://raw.githubusercontent.com/pietro1412/remote-claude-code-bridge/master/scripts/install.ps1" -OutFile "install.ps1"
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\install.ps1
Cosa fa l'installer? Installa Node.js 20+, Git, build tools, Tailscale, Claude Code CLI, PM2 → clona il repo → installa dipendenze → builda tutto → esegue i test → avvia il server → genera il token di accesso → configura l'avvio automatico al boot.
Dopo l'installer L'unica cosa da fare manualmente: accedere a Tailscale con il tuo account (se non l'hai ancora fatto) e salvare il token di accesso che viene stampato alla fine.

3. Configurare Tailscale

Sul PC fisso

# Avvia Tailscale e accedi col tuo account
sudo tailscale up

# Verifica connessione e prendi nota dell'IP
tailscale status
tailscale ip -4     # es. 100.64.0.5

Sul dispositivo remoto (smartphone/portatile)

1
Installa Tailscale: App Store / Google Play / Desktop
2
Accedi con lo stesso account usato sul PC fisso
3
Verifica: dovresti vedere il PC fisso nella lista dispositivi Tailscale

4. Installazione manuale

# Clona il repository
git clone https://github.com/pietro1412/remote-claude-code-bridge.git
cd remote-claude-code-bridge

# Installa dipendenze
npm install

# Copia configurazione
cp .env.example .env

# Build completo
npm run build -w shared
npm run build -w server
npx vite build client

# Esegui test
npm test -w server   # 45 test, tutti verdi

5. Configurazione

Modifica il file .env:

# Server
PORT=3443
NODE_ENV=production

# JWT (auto-generato al primo avvio se vuoto)
JWT_SECRET=

# Tailscale (opzionale, rilevato automaticamente)
TAILSCALE_HOSTNAME=fisso-pietro

# Database
DB_PATH=./data/rccb.db

# Upload foto (max dimensione in MB)
MAX_UPLOAD_SIZE_MB=10

# Logging
LOG_LEVEL=info
Nota: La maggior parte dei valori di default funziona subito. L'unica cosa da cambiare per produzione è NODE_ENV=production.

6. Primo avvio e generazione token

# Avvia con PM2 (consigliato)
pm2 start npm --name "rccb" -- start
pm2 save
pm2 startup   # avvio automatico al boot

# Genera il token di accesso (una sola volta)
curl -X POST http://localhost:3443/api/auth/setup

Risposta:

{
  "access_token": "a1b2c3d4e5f6...lunga-stringa-hex",
  "message": "Save this token securely."
}
IMPORTANTE: salva il token di accesso! Viene generato una sola volta. Ti serve per accedere da tutti i tuoi dispositivi.

7. Connessione da remoto

Da smartphone

1
Assicurati che Tailscale sia attivo sul telefono
2
Apri il browser e vai a: http://NOME-PC:3443 oppure http://100.x.x.x:3443
3
Inserisci il token di accesso e un nome dispositivo (es. "iPhone Pietro")
4
Clicca Accedi — sei dentro!

Android ChromeMenu (⋮) → "Aggiungi a schermata Home"

8. Utilizzo quotidiano

Creare una sessione Claude Code

1
Clicca sul nome nella barra superiore → "+ Nuova Sessione"
2
Inserisci un nome (es. "fantacontratti") e la directory del progetto sul PC fisso
3
Claude Code si avvia e l'output appare nel terminale

9. Manutenzione

Aggiornare RCCB

cd ~/remote-claude-code-bridge
git pull
npm install
npm run build -w shared && npm run build -w server && npx vite build client
pm2 restart rccb

Monitorare

pm2 logs rccb        # Log in tempo reale
pm2 status           # Stato del server
curl http://localhost:3443/api/health   # Health check

Backup

# Database e upload
cp data/rccb.db data/rccb.db.backup
tar czf uploads-backup.tar.gz uploads/

10. Troubleshooting

ProblemaSoluzione
node-pty non si installa Linux: sudo apt install build-essential python3
Windows: installa VS Build Tools con "Desktop development with C++"
Server non raggiungibile da remoto Verifica che Tailscale sia attivo su entrambi i dispositivi: tailscale status
"Authentication required" Il token JWT scade dopo 7 giorni. Rifai il login con il token di accesso originale
Claude Code non si avvia nella sessione Verifica che claude funzioni dal terminale del PC fisso
Upload foto fallisce Controlla che ./uploads/ esista e sia scrivibile. Max 10MB per foto.
PM2 non riparte al boot Esegui pm2 startup e copia/esegui il comando suggerito
Porta 3443 occupata Cambia PORT nel .env oppure: lsof -i :3443 / netstat -ano | findstr 3443
Voice input non funziona Richiede HTTPS o localhost. Su Tailscale funziona se usi i certificati (scripts/tailscale-cert.sh)

11. Architettura di rete

Smartphone (Tailscale) PC Fisso (Tailscale) ┌───────────────┐ ┌──────────────────────┐ │ Browser PWA │ │ RCCB Server (:3443) │ │ React + xterm │◄─── VPN ─────►│ Express + Socket.io │ │ Socket.io │ (WireGuard) │ node-pty → claude │ │ Speech/Camera │ (encrypted) │ SQLite, JWT auth │ └───────────────┘ └──────────────────────┘ Nessuna porta esposta su internet. Tutto il traffico passa dentro la VPN Tailscale (WireGuard).
Sicurezza: Il server si binda automaticamente sull'interfaccia Tailscale (100.x.x.x). Non è raggiungibile da internet. Autenticazione JWT + rate limiting su tutti gli endpoint.
